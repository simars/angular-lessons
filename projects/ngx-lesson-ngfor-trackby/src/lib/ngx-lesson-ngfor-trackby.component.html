Improve *ngFor usability and performance with trackBy

Prevent loss of focus and reconstrution of elements in your *ngFor on collection change with trackBy

Note: “change the collection-data” means to replace the collection with new objects and not keeping the same reference.

Without trackBy :(

Try inputting text in the input boxes

Notice, as the ngFor items change, the text you have input, goes away and the focus is lost

ngFor here is not tracking elements by id or index. When items-collection changes ngFor just destroys and recreates elements for each

  • {{item.id}} | {{item.name}}

With track by :)

Try inputting text in the input boxes

Notice, as the ngFor items change as, text and focus in input boxes is reatained

ngFor is tracking each element to its rendering in dom by id, and it jsut renders the correponding changes in place using using standard change detaction instead of destroying and recreating elements

  • {{itemTrackBy.id}} | {{itemTrackBy.name}}

Note: If you don't have id (unique-key) for each data-item in your collection, you can always tracBy index (second argument pased to tracByFn) If you use a field to track by that is not unique, angualr will trhow an error
